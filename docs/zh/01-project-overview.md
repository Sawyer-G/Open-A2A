# 项目概览

> 设计原则与核心目标见 [00-design-principles.md](./00-design-principles.md)。

---

## 1. 背景与初心：消灭中间商的「自由协议」

### 核心痛点：平台霸权与税收 (Platform Tax)

- 当前的交互（点外卖、约车、找服务等）必须经过中心化平台
- 平台掌控了数据，并从每笔交易中抽取 10%~30% 的佣金
- 平台通过算法操纵消费者的选择

### 愿景：代理人经济 (The Agentic Economy)

- **Agent 间直连**：当 AI 助手普及后，交互应在 Agent 之间直接发生  
  A-Agent ➔ B-Agent ➔ C-Agent（示例：消费者、商家、配送方）
- **零抽成**：去除中转平台，价值在参与者之间 100% 流动
- **数据主权**：AI 助手只服务于其主人，而非平台的竞价排名

### 目标用户

- 全球开源开发者
- Web3 极客
- 研究者

## 2. 产品定位

我们的目标是定义 **「后互联网时代的 TCP/IP 协议」**。我们不卖软件，我们构建规则。

### 核心目标（非具体业务）

Open-A2A 的核心是解决：

- **跨网络 Agent 通信**：不同 Agent 在各自网络下如何发现彼此、建立通信
- **数据交互的形式与逻辑**：消息结构、主题规范、交互模式（广播-收集、请求-响应、委托-确认等）

送餐/外卖是**灵感来源**和**示例场景**，用于验证协议可行性，而非框架要实现的业务目标。

### 设计前提

- **算力便宜**：假设推理成本可接受
- **每用户拥有 Agent**：各参与方运行自己的 AI Agent
- **不重复造轮子**：核心 AI 能力由 [OpenClaw](https://github.com/openclaw/openclaw)、[ZeroClaw](https://github.com/zeroclaw-labs/zeroclaw) 等成熟运行时提供，Open-A2A 专注协议层

### 开源仓库结构预想

| 目录 | 说明 |
|------|------|
| `/spec` | 核心协议文档（Agent 间如何握手、如何报价） |
| `/open_a2a` | 协议的 Python 参考实现（SDK） |
| `/example` | 送餐场景 Demo（用于验证协议，可扩展为其他 domain） |
