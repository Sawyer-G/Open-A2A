# 开发路线图

## MVP 落地步骤

### 第一阶段：Hello Open-A2A

**目标**：在两台电脑（或两个进程）间跑通「广播-响应」。

**任务**：

1. 配置本地 NATS 服务器
2. 编写 `Consumer.py`：发送意图消息
3. 编写 `Merchant.py`：监听到消息后自动回复一个 Offer

---

### 第二阶段：隐私与身份认证 ✅

**目标**：确保只有真实的 Agent 能对话。

**已完成**：

1. ✅ 集成 `did:key`（基于 didlite），消息发送前可选私钥签名，支持 JWS 验签
2. ✅ 偏好存储抽象（`FilePreferencesProvider`），Agent 从 `profile.json` 读取偏好；Solid Pod 接口已预留

---

### 第三阶段：复杂场景模拟 ✅

**目标**：以送餐为示例，验证 A-B-C 多方协作与子任务委托的协议闭环。

**已完成**：

1. ✅ 加入 `Carrier.py`（示例：配送方角色）
2. ✅ 实现模拟支付流（验证结算触发逻辑）
3. ✅ 开源代码库并发布首份 RFC 文档

> 送餐是示例场景，协议设计应保持 domain 无关，便于扩展到其他协作场景。

---

### 未来阶段：多语言 SDK（待定）

**目标**：当项目成熟、有明确需求时，提供多语言参考实现（TypeScript、Go、Java 等）。

**参考**：详见 [07-multi-language-sdk.md](./07-multi-language-sdk.md)。  
**触发条件**：非 Python 技术栈团队接入需求、协议稳定、有社区贡献者维护。
